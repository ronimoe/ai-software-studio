# v0.1 Plan 0 — CI + Homebrew tap stub

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Stand up GitHub Actions CI (lint, typecheck, test on every PR) and create the `ronimoe/homebrew-tap` repo with a stub cask formula, so that distribution infrastructure exists before v0.5 needs it.

**Architecture:** Two independent deliverables, both pure-ops, no app code touched. CI is one workflow file `.github/workflows/ci.yml` running three pnpm scripts on macOS + Linux matrix. The Homebrew tap is a separate GitHub repo (created via `gh repo create`) containing one stub `Casks/ai-software-studio.rb` Ruby cask formula plus a README. Neither deliverable depends on the other.

**Tech Stack:** GitHub Actions, pnpm, Node 20, Rust toolchain, Homebrew Ruby cask DSL, `gh` CLI.

### Task 1: CI workflow file

**Files:**
- Create: `.github/workflows/ci.yml`

- [ ] **Step 1: Confirm the parent directory does not exist yet**

Run: `ls -la .github 2>&1 || echo "missing"`
Expected: `missing` (the directory is created by writing the file below)

- [ ] **Step 2: Create the workflow file**

Write `.github/workflows/ci.yml` with this exact content:

```yaml
name: CI

on:
  pull_request:
    branches: [main]
  push:
    branches: [main]

concurrency:
  group: ci-${{ github.ref }}
  cancel-in-progress: true

jobs:
  frontend:
    name: frontend (${{ matrix.os }})
    runs-on: ${{ matrix.os }}
    strategy:
      fail-fast: false
      matrix:
        os: [ubuntu-latest, macos-latest]
    steps:
      - uses: actions/checkout@v4

      - uses: pnpm/action-setup@v4
        with:
          version: 9

      - uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: pnpm

      - uses: dtolnay/rust-toolchain@stable

      - uses: Swatinem/rust-cache@v2
        with:
          workspaces: src-tauri -> target

      - name: Install Linux Tauri prerequisites
        if: matrix.os == 'ubuntu-latest'
        run: |
          sudo apt-get update
          sudo apt-get install -y \
            libwebkit2gtk-4.1-dev \
            libappindicator3-dev \
            librsvg2-dev \
            patchelf

      - name: Install dependencies
        run: pnpm install --frozen-lockfile

      - name: Generate bindings
        run: pnpm gen:bindings

      - name: Lint
        run: pnpm lint

      - name: Typecheck
        run: pnpm typecheck

      - name: Test
        run: pnpm test
```

- [ ] **Step 3: Validate workflow syntax locally**

Run: `python3 -c "import yaml; yaml.safe_load(open('.github/workflows/ci.yml'))" && echo OK`
Expected: `OK`

- [ ] **Step 4: Commit**

```bash

```

```ruby
cask "ai-software-studio" do
  version :latest
  sha256 :no_check

  url "https://github.com/ronimoe/ai-software-studio/releases/latest/download/ai-software-studio.dmg"
  name "AI Software Studio"
  desc "Local-first command center for AI coding agents (placeholder cask, pre-v0.5)"
  homepage "https://github.com/ronimoe/ai-software-studio"

  # Pre-v0.5 placeholder. The real cask lands when v0.5 ships the first signed-or-stapled DMG.

  app "AI Software Studio.app"

  zap trash: [
    "~/Library/Application Support/AI Software Studio",
    "~/Library/Preferences/studio.aisoftware.app.plist",
    "~/Library/Caches/studio.aisoftware.app",
  ]
end
```

## Shipped deltas (post-execution, 2026-05-18)

The plan above is the *original* design. What actually shipped diverged in a few places. Recording the deltas here so a future reader doesn't reintroduce known bugs.

### Task 1 — CI workflow file

- **Removed `with: version: 9`** from the `pnpm/action-setup@v4` step. The action errors when both `version:` and `packageManager` (in `package.json`) are present; we defer to `package.json` as the source of truth. Fix: `26d27a8`.
- **Added a `Build frontend` step** at the end of the job (`pnpm build`). This produces the real `out/` directory for any downstream consumers and validates the static export still works. Fix: `ebf2a5d`.
- **Briefly added a `Stub frontend dist for binding generation` step** (`mkdir -p out && touch out/.placeholder`) before `Generate bindings` to work around `tauri::generate_context!()` validating `frontendDist` at compile time. Fix: `ebf2a5d`. **Later removed** (`1ad0f68`) once the root cause was fixed in Rust — see "Followup cleanup" below.

### Task 3 — Verify CI on a real PR

- **Step 4 (`gh pr close --delete-branch`) was not executed.** The plan's Step 4 note already allowed skipping when not on a throwaway branch. The branch `plan-0-ci-and-tap` contained real work, so the PR (#1) was instead merged into `main` once CI was green. Merge commit: `623a574`. The smoke-test PR title (`"ci: smoke test (close without merging)"`) is frozen in `main`'s history as a small wart.

### Task 5 — Cask formula

- **Added `depends_on macos: :big_sur`** between the comment block and the `app` stanza. `brew style` enforces `Homebrew/OSDependsOn` for any cask with an `app` stanza. Fix: `29840a3`.
- **Used the bare-symbol form `:big_sur` instead of the string form `">= :big_sur"`.** The `:big_sur` symbol in cask DSL is interpreted as "Big Sur or later" (a minimum, not an exact match). The string form fails `brew style`.
- **Reordered the `zap trash:` array alphabetically** (`Application Support` → `Caches` → `Preferences`). `brew style` enforces `Cask/ArrayAlphabetization`. Fix: `29840a3`.

### Followup cleanup (commits `756b4c7`, `1ad0f68`)

- **Gated `pub fn run()` in `src-tauri/src/lib.rs` behind `#[cfg(not(test))]`** so `tauri::generate_context!()` no longer fires during `cargo test` builds. This eliminates the chicken-and-egg between `gen:bindings` and the `out/` directory at the source. Commit: `756b4c7`.
- **Added `--lib` to the `gen:bindings` script in `package.json`** (`cargo test --lib --manifest-path src-tauri/Cargo.toml export_bindings -- --nocapture`). Without `--lib`, `cargo test` also compiles the binary target, which still depends on `run()` and therefore on `out/`. Restricting to `--lib` ensures binding export works on a fresh clone. Commit: `756b4c7`.
- **Removed the `Stub frontend dist for binding generation` step** from `.github/workflows/ci.yml` once the Rust fix made it redundant. Commit: `1ad0f68`.
- **Known cosmetic warning:** `cargo check --lib --tests` now emits an `unused_imports` warning on `use state::AppState;` in `src-tauri/src/lib.rs:18` because `run()` (the sole consumer) is gated out during test builds. Production (bin) builds remain clean.
